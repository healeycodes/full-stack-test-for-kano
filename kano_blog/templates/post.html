Why I Love WebSockets

WebSockets have been aiding real time communication on the web for a few years now and the technology is really starting to mature. With easy to use libraries like socket.io that fallback to HTTP polling for old browsers, and ultra performant implementations like uWebSockets which focus on low user space memory, developers are spoilt for choice.

On uWebSockets, one million connections require just ~111mb of user space memory (104 bytes per WebSocket!)

Creating real time web applications has never been easier, with tiny chat applications only requiring ~25 total lines of code on the front- and back-end of a Node.js setup. Check out the barebone back-end of socket.io's introductory chat application:

Multiplayer games like agar.io rely on this technology to broadcast the location of millions of squishy blobs each and every day. Instead of raw HTTP requests with extra data such as cookies and other headers, and potentially a client authentication per each request, the duplex connection is headerless and WebSockets only authenticates during the handshake. It's even efficient enough to control quadcopters.
